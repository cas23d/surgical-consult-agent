How It Works

1
Consult page arrives

Resident receives consult with patient MRN and reason

2
Agent pulls chart via FHIR

Vitals, labs, imaging, meds, notes — all pulled automatically

3
AI analyzes the case

Triage, red flags, gaps in management, evidence-based plan

4
Outputs ready

Consult note, staffing summary, and follow-up tasks

Design Principles

Eyes and ears, not autopilot

The agent surfaces what the resident might miss at 2 AM — it doesn't make decisions. The clinician stays in the loop.

Guideline-transparent

Every recommendation cites the specific guideline and society. When a guideline doesn't apply cleanly, the agent says so.

Built for messy reality

Residents type in shorthand. Chart data is incomplete. Radiology reads can be wrong. The agent handles all of it.

No frameworks, no magic

Raw API calls to Claude via the Anthropic SDK. FHIR R4 for EHR data. Every line of code is readable and auditable.

Why I Built This

I'm a physician who left clinical practice to work at the intersection of medicine and AI. Not to advise from the sidelines — to build.

This agent exists because I've been the resident at 2 AM staring at a consult page, pulling together labs, imaging, and vitals from six different screens while trying to think clearly about a sick patient. That experience is the spec. The clinical reasoning in this demo isn't prompted — it's how I actually think through these cases.

I built this with Python, the Anthropic API, and FHIR R4 — no templates, no drag-and-drop. Every clinical decision, every guideline citation, every edge case flag reflects real surgical training.
